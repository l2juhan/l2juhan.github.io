## AWS Billing Console

- AWS 계정에서 발생하는 **모든 비용 관련 활동을 확인, 관리, 분석, 지불**할 수 있는 **중앙 허브**
- 이곳은 단일 서비스가 아니라, **비용 탐색기(Cost Explorer), AWS 예산(Budgets)** 등 여러 비용 관리 도구들을 한 곳에 모아놓은 **포털**

## 주요 기능

- Billing Console에 접속하면 다음과 같은 주요 기능들을 사용할 수 있습니다.

### 대시보드 (Dashboard)

- **핵심 역할:** 현재 비용 상태를 한눈에 파악하는 첫 화면.
- **무엇을 볼 수 있나요?**
- 현재까지의 월별 누적 요금
- 이번 달 예상 청구액
- 지난달 대비 비용 증감 추이
- 서비스별(EC2, S3 등) 비용 지출 상위 5개 항목

### 청구서 (Bills)

- **핵심 역할:** 월별 청구서(인보이스)를 확인하고 다운로드하는 곳.
- **무엇을 할 수 있나요?**
- 매월 발행된 청구서를 상세 내역별로 조회할 수 있습니다.
- PDF 형식의 공식 청구서를 다운로드하여 회계 처리에 사용할 수 있습니다.

### 비용 탐색기 (Cost Explorer)

- **핵심 역할:** 과거 및 현재 비용을 심층적으로 분석하고 시각화하는 도구.
- **무엇을 할 수 있나요?**
- 지난 12개월간의 비용 데이터를 그래프로 탐색할 수 있습니다.
- 서비스, 리전, 태그 등 다양한 필터를 적용하여 "왜 이번 달 비용이 늘었을까?" 와 같은 질문의 원인을 분석할 수 있습니다.

### AWS 예산 (AWS Budgets)

- **핵심 역할:** 예산 임계값을 설정하고, 초과 시 경고 알림을 받는 서비스.
- **무엇을 할 수 있나요?**
- "이번 달 총비용이 $1,000를 넘으면 나에게 이메일을 보내줘" 와 같이 예산을 설정합니다.
- 비용이 예산을 초과하거나, 초과할 것으로 '예측'될 때 알림을 받아 '요금 폭탄'을 사전에 방지할 수 있습니다.

### Savings Plans / 예약 인스턴스 (Reserved Instances)

- **핵심 역할:** 장기 약정 할인을 구매하고 관리하는 곳.
- **무엇을 할 수 있나요?**
- 1년 또는 3년 약정 사용을 통해 EC2, RDS 등의 요금을 크게 할인받는 Savings Plans나 예약 인스턴스를 구매하고, 보유 현황을 관리합니다.

### 비용 할당 태그 (Cost Allocation Tags)

- **핵심 역할:** 리소스에 붙인 태그를 비용 분석에 사용하도록 활성화하는 곳.
- **무엇을 할 수 있나요?**
- `Project`, `Department` 등 사용자가 직접 만든 태그를 활성화하여, 비용 탐색기나 예산에서 "A 프로젝트에서 발생한 비용은 얼마인가?"를 추적할 수 있도록 설정합니다.

## 접근 권한에 대한 중요 노트

- 기본적으로 AWS 계정을 처음 만든 **루트 사용자(Root User)만 Billing Console의 모든 정보에 접근**할 수 있습니다.
- 일반 IAM 사용자가 결제 정보에 접근하게 하려면, **루트 사용자가 '내 계정' 설정에서 'IAM 사용자의 결제 정보 접근' 옵션을 수동으로 활성화**해야 합니다. 이 과정을 거치지 않으면 IAM 사용자는 비용 관련 페이지를 볼 수 없습니다.
