AWS SAA 시험에서 중요도가 높은(★★★) 네트워킹 서비스를 정리합니다.

## VPC와 서브넷

- **VPC** — 가상의 네트워크 공간 (컴퓨터들끼리 서로 통신하는 공간)
- **서브넷** — VPC를 작은 네트워크 단위로 나눈 것
  - **퍼블릭 서브넷**: 외부에서 접근 가능
  - **프라이빗 서브넷**: 외부에서 접근 불가

## 인터넷 게이트웨이

VPC와 외부 인터넷 간에 통신할 수 있게 해주는 장치입니다. VPC에 연결하면 해당 서브넷이 퍼블릭 서브넷이 됩니다.

> 외부 인터넷 ↔ VPC

## NAT 게이트웨이

프라이빗 서브넷에서 외부 인터넷으로만 나갈 수 있는 **단방향 출입구**입니다. 외부에서 서브넷으로는 접근 불가합니다.

> 프라이빗 서브넷 → 외부 인터넷 (역방향 불가)

NAT 게이트웨이는 **퍼블릭 서브넷에 연결**합니다. 프라이빗 서브넷을 외부에 직접 노출시키지 않으면서도 인터넷 접근을 가능하게 하기 위함입니다.

## VPC Endpoint

인터넷을 거치지 않고 **AWS 서비스(S3, DynamoDB 등)에 직접 연결**하는 서비스입니다.

| 유형 | 대상 |
|---|---|
| 게이트웨이 VPC 엔드포인트 | S3, DynamoDB |
| 인터페이스 VPC 엔드포인트 | S3, DynamoDB 제외 모든 AWS 리소스 |

## PrivateLink

인터넷을 거치지 않고 **다른 VPC의 애플리케이션에 접속**하게 하는 서비스입니다.

## VPC 피어링

서로 다른 VPC를 **사설 네트워크로 직접 연결**하는 기능입니다.

## 비교 정리

| 서비스 | 용도 | 연결 방향 |
|---|---|---|
| **NAT Gateway** | 외부 인터넷 연결 | Private EC2 → 인터넷 |
| **VPC Endpoint** | AWS 서비스 연결 | VPC → AWS 서비스 (인터넷 X) |
| **PrivateLink** | 애플리케이션 연결 | VPC → 다른 VPC 앱 (인터넷 X) |
| **VPC 피어링** | VPC 연결 | VPC ↔ VPC (인터넷 X) |

| 상황 | 추천 |
|---|---|
| VPC 2개를 간단히 연결 | VPC 피어링 |
| 연결할 VPC가 많고 복잡 | Transit Gateway |
| 특정 서비스만 안전하게 공유 | PrivateLink |
